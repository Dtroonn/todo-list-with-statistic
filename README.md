# Todo-list-with-statistics запуск

0. Preview: https://todo-list-with-statistic.netlify.app/ (может потребоваться иностранный vpn т.к некоторые провайдеры блокируют netlify)
1. Установить зависимости с помощью npm ci
2. Запустить с помощью команды: npm run build-and-preview. Если по каким-то причинам не выполнится,
   то сначала запустить npm run build, а затем npm run preview.

## Возможные дальнейшие улучшения и оптимизации

1. При разрастании проекта уйти от монолита к архитектуре микрофронтендов. Например уже сейчас
   страницы с задачами и статистикой могут быть отдельными микрофронтами, что откроет возможность независимых релизов частей приложения.(для этого нужно будет перейти на webpack т.к переход будет на webpack module federation)

2. Перейти на FSD, если команда согласится(предполагается что далее проект будет разрабатываться в команде) т.к вокруг методологии ходят споры. Но на FSD есть смысл переходить если мы остаемся в монолите, либо внедрять его в объемные микрофронты.

3. Если очень важна производительность на слабых устройствах/мобильных устройствах, то можно рассмотреть переход на next.js, посредством которого за счет react server components и partial rendering можно будет поднять TTI, LCP метрики. Тут упоминается именно next.js, потому что он остается наиболее зрелым и готовым к продакшену фреймворком для использования react server components.

4. Если приложению SSR не нужен, то можно написать свое кастомное решение с RSC, но нужен будет переход на webpack и react-server-dom-webpack. Это самое затратное решение т.к это не распространенный подход + придется имплементировать свой node сервер самостоятельно.

5. Подключить i18n т.к в будущем может появиться мультиязычность и лучше сразу хранить значения в словаре.

6. Доработать A11y(Accessibility) и добавить адаптивность

7. Подключить React Compiler

8. Перейти на css modules

9. Добавить Service Worker для продвинутого кеширования
